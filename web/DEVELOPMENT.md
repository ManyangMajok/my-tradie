# Local Development Guide

Get from `git clone` to a running dev environment in under 30 minutes.

---

## Prerequisites

Install these before starting. Versions matter — the project targets production Ubuntu 24.04 LTS
with these exact versions.

| Tool | Minimum version | Notes |
|---|---|---|
| PHP | 8.4+ | Ensure `php` on PATH. Windows: [php.net/downloads](https://windows.php.net/download/) |
| Composer | 2.x | [getcomposer.org](https://getcomposer.org/download/) |
| PostgreSQL | 16+ | On Windows, installer defaults to port **5432**; this project uses **5433** — set it during install or configure below |
| Redis | 7+ | Windows: use [Memurai](https://www.memurai.com/) (Redis-compatible, free for dev). Default port 6379 |
| Node.js | 20+ | LTS recommended. [nodejs.org](https://nodejs.org/) |
| Git | any | |

---

## 1. Clone the repo

```bash
git clone <repo-url>
cd my-tradie
```

The Laravel application lives in `web/`. All commands below run from inside `web/` unless noted.

```bash
cd web
```

---

## 2. Install PHP dependencies

```bash
composer install
```

> **Note:** `composer.json` has `"platform-check": false`. This is intentional — Horizon requires
> `ext-pcntl` and `ext-posix`, which are Linux/macOS-only extensions. Disabling the platform check
> lets Composer install on Windows without errors. The extensions exist on the production server
> (Ubuntu 24.04), so this is safe. See [Queue worker](#6-start-the-dev-stack) below.

---

## 3. Environment file

```bash
cp .env.example .env
php artisan key:generate
```

Then open `.env` and configure the following sections:

### Database

```dotenv
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5433          # adjust if your PostgreSQL uses 5432
DB_DATABASE=my_tradie
DB_USERNAME=my_tradie
DB_PASSWORD=your_db_password_here
```

### Redis

```dotenv
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
```

### Reverb (WebSockets — values generated by reverb:install, already in .env.example)

```dotenv
REVERB_APP_ID=your_reverb_app_id
REVERB_APP_KEY=your_reverb_app_key
REVERB_APP_SECRET=your_reverb_app_secret
REVERB_HOST="localhost"
REVERB_PORT=8080
REVERB_SCHEME=http
```

## 4. Create the databases

Connect to PostgreSQL as the superuser and create two databases:

```sql
CREATE DATABASE my_tradie OWNER my_tradie;
CREATE DATABASE my_tradie_testing OWNER my_tradie;
```

From a terminal (adjust port and superuser password as needed):

```bash
psql -U postgres -h 127.0.0.1 -p 5433 -c "CREATE DATABASE my_tradie OWNER my_tradie;"
psql -U postgres -h 127.0.0.1 -p 5433 -c "CREATE DATABASE my_tradie_testing OWNER my_tradie;"
```

If the `my_tradie` role doesn't exist yet:

```sql
CREATE ROLE my_tradie WITH LOGIN PASSWORD 'your_password_here';
```

## 5. Run migrations

```bash
php artisan migrate
```

Verify with:

```bash
php artisan migrate:status
```

All migrations should show `Ran`.

## 6. Install JS dependencies and build assets

```bash
npm install
npm run build
```

The build output goes to `public/build/`. For development you'll use `npm run dev` instead
(see next section).

---

## 7. Start the dev stack

Open four terminals, all from `web/`:

| Terminal | Command | Purpose |
|---|---|---|
| 1 | `php artisan serve` | Laravel HTTP on `http://127.0.0.1:8000` |
| 2 | `npm run dev` | Vite HMR (hot reload) on `:5173` |
| 3 | `php artisan queue:work` | Queue worker — see Windows note below |
| 4 | `php artisan reverb:start` | WebSocket server on `:8080` |

Or use the Composer `dev` script (runs Laravel, queue, log watcher, and Vite together — excludes Reverb):

```bash
composer run dev
```

Visit `http://127.0.0.1:8000` — should show the Tradify welcome page.

### Queue worker — Windows vs Linux

**Windows (local dev):** `php artisan queue:work`

**macOS / Linux / production:** `php artisan horizon`

Horizon requires `ext-pcntl` and `ext-posix`, Linux/macOS-only PHP extensions that don't exist
on Windows. `queue:work` processes jobs correctly on Windows; you just won't have Horizon's
supervisor metrics populated locally.

The Horizon dashboard at `/horizon` loads on all platforms regardless — it just shows no active
workers when running on Windows.

## 8. Verify the setup

```bash
# Tests (run against my_tradie_testing database)
php artisan test

# Code formatting check
./vendor/bin/pint --test

# Production build check
npm run build
```

All three should pass with zero errors before you write any code.

### Confirm Redis is reachable

```bash
php artisan tinker
>>> Cache::put('ping', 'pong', 10)
>>> Cache::get('ping')   # should return 'pong'
```

### Confirm PostgreSQL is reachable

```bash
php artisan tinker
>>> DB::select('SELECT version()')
```

## 9. Common issues

**`SQLSTATE[08006]` PostgreSQL connection refused**
Check `DB_PORT` in `.env`. PostgreSQL on this project uses port `5433` (not the default `5432`).

**`Connection refused` on Redis**
Check Memurai (Windows) is running. Defaults to port `6379`.

**`php artisan horizon` fails with pcntl error**
Expected on Windows. Use `php artisan queue:work` instead. See §7 above.

**Vite assets not loading (`http://localhost:5174/` in console)**
Port `5173` was already in use; Vite took `5174`. This is fine — the Vite plugin auto-detects
the port. Refresh the browser.

**`npm run build` fails with TypeScript errors**
Run `npx tsc --noEmit` to see the type errors. Fix before committing.
